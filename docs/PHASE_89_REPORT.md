# Phase 89 Report: One-Shot Evidence Reporting

生成日期：2026-07-09

## Goal

把 Phase 88 手动拼装的 one-shot closure evidence 产品化，让 Migration Guard 可以从 baseline、latest run、compare artifact 和 target git status 自动生成一份 one-shot readiness/reporting artifact。

## Delivered

New CLI command:

```text
migration-guard one-shot report
```

Key options:

```text
--baseline <path>
--current <path>
--compare <compare.json>
--max-source-file-delta <n>
--skip-target-git
--strict
--json
```

New implementation:

```text
src/core/oneShot.ts
src/core/oneShot.test.ts
```

Updated entry points:

```text
src/cli.ts
package.json
README.md
```

The report checks:

- latest baseline snapshot is available
- latest verification run is available
- critical checks passed
- behavior probes passed
- baseline/current compare passed
- source file count delta stays within the explicit budget
- target repository is clean unless skipped

Output artifacts:

```text
<artifactsDir>/one-shot/one-shot-report-<timestamp>.json
<artifactsDir>/one-shot/one-shot-report-<timestamp>.md
```
